How It Works

Four Steps to Clean PRs

PR-Surgeon integrates seamlessly into your git workflow. Just one command before you push.

01

Scan Your Diff

Run pr-surgeon scan and it automatically reads your git diff against the target branch. Detects all changed files, insertions, deletions, and content.

02

Analyze Complexity

Each file is analyzed for cognitive complexity (nesting, control flow, callbacks), domain classification (DB/API/UI/Auth), import dependencies, and risk level.

03

Generate Split Plan

The smart splitter groups files by domain, respects import coupling within domains, assigns merge order (DB first, UI last), and calculates per-PR metrics.

04

Review & Act

Get a beautiful terminal report or a full HTML X-Ray diagnostic. See before/after metrics and follow the split suggestions to ship clean, focused PRs.

Scoring System

Multi-Dimensional Complexity Score

Five independent metrics, each scored 0-300, combined with research-backed weights to give a single actionable score.

Metric Weight What It Measures
📏 Size 25% Lines changed, files touched, average changes per file
🧠 Cognitive 25% Nesting depth, control flow, callbacks, promise chains
🌎 Domain Spread 20% How many layers (DB/API/UI/Auth/Config) are touched
🔗 Coupling 15% Import dependencies between changed files
⚠️ Risk 15% Critical paths: auth, security, DB migrations

Severity Levels

Score Severity What It Means
0 – 50 ✔ CLEAN Ship it. Reviewers will love you.
51 – 100 ⚠ MODERATE Manageable. Keep an eye on it.
101 – 150 🔶 COMPLEX Consider splitting. Review quality at risk.
151 – 250 🚨 MONSTER PR Split immediately. Reviewers will hate this.
251 – 300 💀 CATASTROPHIC Emergency intervention needed.
Get Started

Up and Running in 30 Seconds

No configuration. No API keys. Just clone, install, and scan.

⚡ Quick Install

# Clone the repo
$ git clone https://github.com/hamza0281/pr-surgeon.git
$ cd pr-surgeon
$ npm install

# Run a demo (no git repo needed)
$ npm run demo

🔍 Scan Your Code

# Scan against main branch
$ node src/cli.js scan

# Scan against a specific branch
$ node src/cli.js scan -b develop

# Generate HTML X-Ray report
$ node src/cli.js scan --report

📈 More Commands

# Analyze git history patterns
$ node src/cli.js history

# Install pre-push hook
$ node src/cli.js hook install

# Full HTML report only
$ node src/cli.js report

🛠 Programmatic API

// Use in your own tools
import { scanCurrentDiff,
  analyzeComplexity,
  suggestSplits
} from 'pr-surgeon';

const diff = await scanCurrentDiff('main');
const report = analyzeComplexity(diff);
const splits = suggestSplits(report);
